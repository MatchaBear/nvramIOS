Selamat Datang di Halaman Dasar-Dasar Cisco IOS

Jadi, si NVRAM itu adalah si memory card.
Ketika booting up, isi NVRAM di-copy ke RAM. Si router/switch nge-run yg ada di RAM.
Ketika saving config (write mem / copy run start), config yg ada di RAM (karna dia yg di-run kan) di-copy ke NVRAM (start).
Ketika hapus config (write erase / hapus memori), config yg ada di NVRAM yg dihapus.

Jadi ketika mau reload switch: kalo di-prompt, jangan Save! Kalo di-save, kan, copy yg ada di RAM ke NVRAM lagi, jadi yg tadinya NVRAM kosong, jadi ada isinya lagi..!!
